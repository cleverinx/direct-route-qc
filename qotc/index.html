Hero & Homepage

4 issues
#01Critical

Season 2 Trailer modal opens but video does not play

"Watch Season 2 Trailer" opens a modal but the video fails to load or play. The modal renders empty.

HeroVideo / Modal
RecommendationVerify the video embed URL is correctly wired to the modal. Confirm hosting permissions and that the iframe allow attributes include autoplay.
#02High

Main nav items bunch together on mid-size desktop viewports (~1100–1280 px)

Navigation items crowd and visually clip at intermediate widths before the mobile breakpoint triggers.

NavigationResponsive
RecommendationAdd an intermediate breakpoint that collapses secondary items or switches to the hamburger menu before the current mobile threshold.
#03High

"Vote / Interact" and "More To Do" both resolve to the same page; hashtag renders as ##

Both Quick Links entries navigate to the same destination. The hashtag label also double-renders as ##QUEENOFTHECASTLE.

NavigationQuick Links
RecommendationDifferentiate routes or remove the duplicate. Fix the double-hash — likely a template variable being escaped twice.
#04Medium

Episode "Watch" CTAs are small text — streaming intent is not apparent

On the Style Roundup list "Watch the Episode" appears as a small text link. Users may not realise they can stream directly from the site.

EpisodesCTA
RecommendationReplace with a pill button or icon-play button consistent with the hero CTAs. Hit target should be at least 44 × 44 px.

Backend & Data Integrity

3 issues
#26Critical

Forms may display a "success" state even when the database save failed

Contest and watch-party forms mark submission as complete after the request is sent, without verifying that the Supabase insert actually succeeded. Only honeypot bot blocks are checked. A silent DB error produces a false "done" state — the entry is lost with no error surfaced to the user.

FormsData IntegritySupabase
RecommendationAwait the Supabase insert response and inspect the returned error object before setting success state. If the insert fails, surface a user-facing error and retain form data so the user can retry. Log the error server-side.
#27High

Spam / bot protection is incomplete — three gaps identified

Some protections are in place but the Royal Decree form is missing a honeypot, and no CAPTCHA or IP rate limiting is applied across any form.

Protection measureStatus
One vote per poll per browser (DB + localStorage)Done
Honeypot on contest & watch-party formsDone
Honeypot on Royal Decree formMissing
CAPTCHA (Turnstile or equivalent)Missing
IP rate limiting (server-side or Cloudflare)Missing
FormsSecurityRoyal Decree
RecommendationAdd a honeypot field to the Royal Decree form immediately. Evaluate Cloudflare Turnstile (invisible CAPTCHA) on all submission forms before launch. Implement server-side rate limiting at the Supabase edge function layer or via Cloudflare rules.
#28High

No transactional email system — copy promises inbox delivery that isn't implemented

UI copy (e.g. "Watch your inbox") implies users will receive confirmation or welcome emails. No transactional email integration exists — entries are stored in Supabase only. If collected emails are later exported to a sending service, providers may reject or flag the list because there is no confirmed opt-in trail.

EmailFormsContests
Recommendation(1) Confirm with the team whether transactional email is handled outside the site. If users expect automatic confirmations, integrate a sending service (e.g. Resend, Postmark, Mailchimp Transactional) before launch. (2) Implement double opt-in or a clear consent checkbox at the point of email capture so the list is deliverable and legally defensible. (3) Remove or update any UI copy implying an email will be sent until the system is in place.
